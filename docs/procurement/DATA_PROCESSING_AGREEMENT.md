## 1. DEFINITIONS

**"Personal Data"** means any information relating to an identified or identifiable natural person processed by Seizn on behalf of Customer.

**"Processing"** means any operation performed on Personal Data, including collection, storage, retrieval, use, disclosure, or deletion.

**"Subprocessor"** means any third party engaged by Seizn to process Personal Data on Customer's behalf.

**"Data Subject"** means the identified or identifiable natural person to whom Personal Data relates.

**"AI Memory Data"** means conversational data, user preferences, and contextual information stored in Seizn's memory system.

---

## 2. SCOPE AND PURPOSE

### 2.1 Subject Matter

This DPA governs Seizn's processing of Personal Data on behalf of Customer in connection with the Seizn platform services.

### 2.2 Nature and Purpose

Seizn processes Personal Data to:
- Store and retrieve AI conversation memories
- Provide AI governance and compliance features
- Enable AI agent tool execution
- Generate analytics and reporting

### 2.3 Categories of Data Subjects

- Customer's employees and contractors
- Customer's end users
- Other individuals whose data Customer submits to Seizn

### 2.4 Types of Personal Data

- User identifiers (user ID, email, name)
- Conversation content and context
- Usage data and preferences
- AI interaction logs

## 3. CUSTOMER OBLIGATIONS

### 3.1 Lawful Basis

Customer warrants that it has a lawful basis for processing Personal Data and has provided appropriate notices to Data Subjects.

### 3.2 Instructions

Customer's instructions to Seizn are documented in:
- This DPA
- The Service Agreement
- Configuration settings in the Seizn platform
- Written requests to Seizn support

### 3.3 Data Accuracy

Customer is responsible for the accuracy, quality, and legality of Personal Data submitted to Seizn.

---

## 4. SEIZN OBLIGATIONS

### 4.1 Processing Limitations

Seizn shall:
- Process Personal Data only on Customer's documented instructions
- Not process Personal Data for any other purpose
- Notify Customer if required by law to process data otherwise

### 4.2 Confidentiality

Seizn ensures that personnel authorized to process Personal Data:
- Are bound by confidentiality obligations
- Receive appropriate data protection training
- Process data only as instructed

### 4.3 Security Measures

Seizn implements technical and organizational measures including:

**Technical Measures:**
- Encryption at rest (AES-256) and in transit (TLS 1.3)
- Multi-tenant isolation via Row-Level Security
- Access controls and authentication
- Intrusion detection and monitoring
- Regular security testing

**Organizational Measures:**
- Security policies and procedures
- Employee background checks
- Security awareness training
- Incident response procedures
- Business continuity planning

### 4.4 Subprocessors

Seizn maintains a list of authorized subprocessors at [URL].

Seizn shall:
- Provide 30 days' notice of new subprocessors
- Ensure subprocessors are bound by equivalent data protection obligations
- Remain liable for subprocessor compliance

Customer may object to new subprocessors within 14 days of notice.

### 4.5 Data Subject Rights

Seizn shall assist Customer in responding to Data Subject requests for:
- Access to Personal Data
- Rectification of Personal Data
- Erasure of Personal Data (Right to be Forgotten)
- Data portability
- Restriction of processing
- Objection to processing

### 4.6 Data Breach Notification

Seizn shall notify Customer without undue delay (within 72 hours) upon becoming aware of a Personal Data breach, including:
- Nature of the breach
- Categories of data affected
- Approximate number of Data Subjects affected
- Likely consequences
- Measures taken or proposed

### 4.7 Deletion and Return

Upon termination or Customer request:
- Seizn shall delete or return all Personal Data
- Deletion shall be certified in writing upon request
- Standard retention is 30 days post-termination
